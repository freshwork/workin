Release Notes for IronEye SQL, Version 1.2

August 8, 2003

This document describes the following aspects of IronEye SQL, Version 1.2:

  • New IronEye SQL Features
  • System Requirements
  • Directory Sructure
  • Installing IronEye SQL

New IronEye SQL Features

IronEye SQL can track all the SQL that flows between an application and a database through a GUI console, log files, or both. IronEye SQL provides the following new features in this release:

  • Enhanced user interface features that provide easier access to the IronEye SQL functionality.
  • Improved functionality for the Show Bound Variables feature, which provides a way for you to dynamically show SQL bound variable statements in the user interface without having to restart the system
  • Enhanced Ant integration.

System Requirements

IronEye SQL requires the following minimum disk space and memory:

  • Disk Space � 5MB
  • Memory � 120MB

Note: Logging is enabled by default, and the IronEye SQL log file, spy.log, can get very large. To accommodate the growth of spy.log, it is recommended that you reserve significant disk space beyond these minimum requirements.
