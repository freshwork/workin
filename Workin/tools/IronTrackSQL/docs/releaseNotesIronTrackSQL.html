IronTrack SQL Features

Building on the discrete performance testing capabilities of IronEye SQL, IronTrack SQL increases the visibility of performance problems from build to build through time-based views of JDBC performance metrics. IronTrack SQL easily enables Java performance tuning by:

  • Displaying database usage over time for all SQL queries in any JDBC application, which can identify problems from query interaction to discovering which queries coincide with spikes in database usage or slowness.
  • Overlaying and comparing performance datasets to give historical analysis and immediately identify performance degradation.
  • With no source code changes, easily embedding performance testing for entire development teams to facilitate test- driven development and continuous performance.

With IronTrack SQL, you can:

  • Uncover performance degradation over time by tracking and storing historical JDBC performance data.
  • Compare performance metrics from multiple datasets to find and fix performance problems faster.
  • Automate performance testing via Ant-based assertions, proactively alerting the development team to performance degradations.
  • Establish baseline performance metrics against which automated performance testing can be compared.
  • Create performance history, allowing teams to analyze longer-range views of performance trends.

System Requirements

IronTrack SQL requires the following minimum disk space and memory:

  • Disk Space � 5MB
  • Memory � 120MB

Note: Logging is enabled by default, and the IronTrack SQL log file, spy.log, can get very large. To accommodate the growth of spy.log, it is recommended that you reserve significant disk space beyond these minimum requirements.
